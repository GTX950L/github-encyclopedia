# 🔒 安全披露政策

感谢你对 **GitHub 使用百科全书** 项目的关注！

## 📧 报告安全漏洞

**请不要通过公开的 Issue 或 Discussion 报告安全漏洞！**

请直接通过 GitHub 的 **Private vulnerability reporting** 功能提交：

> 仓库页面 → Settings → Security → Private vulnerability reporting

如果你的 GitHub 账号没有此功能，可以发送邮件到项目的 Issues 中说明情况（注意不要透露漏洞细节）。

## ⏳ 响应时间

- 确认收到：72 小时内
- 初步评估：1 周内
- 修复发布：根据严重程度

## 💡 负责任的披露

我们感谢你负责任地披露漏洞。在漏洞修复前，请不要公开讨论漏洞细节。

---

> 本项目是一个**文档项目**，安全风险较低。主要需要关注的是：
> - 外部链接是否被劫持
> - 自述文件中的联系信息是否泄露
> - CI/CD 配置是否被篡改
