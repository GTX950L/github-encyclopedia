## 🎯 项目简介

本项目是一个**完全免费、开源的 GitHub 使用指南**，旨在帮助所有人从编程新手成长为 GitHub 资深用户。

无论你是：
- 🔰 **编程新手**：刚学会写代码，不知道如何保存和分享
- 🌱 **GitHub 初学者**：注册了账号，但不懂各个按钮的作用
- 🌿 **进阶用户**：想学习团队协作、CI/CD 等高级功能
- 🌳 **资深开发者**：想系统了解 GitHub 的所有特性

这里都有适合你的内容！

## 📖 内容结构

本项目按照**学习路径**组织为 **30 章 + 5 个附录**（约 20 万字），覆盖从新手到资深玩家的全流程。

| 阶段 | 章节 | 定位 |
|------|------|------|
| 🚀 **新手入门** | 第1-4章 | 认识 GitHub、注册、界面、第一个仓库 |
| 🌟 **基础操作** | 第5-8章 | Clone、Commit、Branch、Merge |
| 🤝 **团队协作** | 第9-12章 | Fork、PR、Issues、Projects、Wiki |
| ⭐ **社交功能** | 第13-16章 | Star、Follow、README、开源协议 |
| 🤖 **AI 与自动化** | 第17-20章 | Token、AI 自动化、Actions、Pages |
| 🔒 **网络安全** | 第21-24章 | 安全问题、SSH、2FA、密钥管理 |
| 🎓 **资深玩家** | 第25-28章 | Git 高级、CLI、API、最佳实践 |
| ☁️ **云端与社区** | 第29-30章 | Codespaces、Discussions |
| 📎 **附录 A-E** | 项目结构、Git 命令速查、Actions 市场、Jekyll 主题、Conventional Commits |

> 📑 **完整章节索引**请查看 [docs/index.md](docs/index.md)（在线版首页）

## ❓ 常见问题

**Q: 我只是想下载别人的代码，需要学这么多吗？**  
A: 不需要！直接看 [第5章：下载代码](docs/chapter05-download-code.md) 就够了。

**Q: 我想让别人也能用我的代码，该怎么办？**  
A: 看 [第4章：第一个仓库](docs/chapter04-first-repository.md) 和 [第16章：开源协议](docs/chapter16-license-choice.md)。

**Q: Token 是什么？为什么需要它？**  
A: 看 [第17章：GitHub Token](docs/chapter17-token-guide.md)，这是让 AI 或脚本访问你账号的关键。

**Q: 我觉得某个地方写错了，怎么办？**  
A: 太好了！请在 [Issues](https://github.com/GTX950L/github-encyclopedia/issues) 中提出，或者直接提交修改。
