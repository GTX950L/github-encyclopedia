## 注册 GitHub 账号

### 步骤详解

#### 第1步：访问 GitHub 官网

```
在浏览器中打开：https://github.com/
```

#### 第2步：开始注册

```
点击首页右上角的 "Sign up" 按钮
│
填写信息：
├── Email address（邮箱地址）
│    → 建议使用常用邮箱（QQ、163、Gmail 等）
│
├── Password（密码）
│    → 至少 15 个字符，或 8 个字符（含数字和小写字母）
│    → 建议：使用密码管理器生成强密码
│
└── Username（用户名）
     → 这是你的唯一标识！
     → 以后你的仓库地址就是：github.com/你的用户名
     → 建议：简短、好记、专业
```

> 💡 **用户名建议**：
> - ✅ 推荐：`GTX950L`、`zhang-san`、`dev123`
> - ❌ 避免：`asdfghjkl`、`woaini1314`、`fuckcode`
> - 以后可以改用户名，但会留下重定向，建议一开始就想好。

#### 第3步：验证邮箱

```
1. GitHub 会向你的邮箱发送验证邮件
2. 打开邮箱，找到来自 GitHub 的邮件
3. 点击邮件中的验证链接
4. 验证成功！
```

> ⚠️ **收不到验证邮件？**
> - 检查垃圾邮件箱
> - 等待 5-10 分钟
> - 在 GitHub 上点击 "Resend verification email"

## 配置个人资料

### 进入个人设置

```
方法1：点击右上角头像 → Settings
方法2：直接访问 https://github.com/settings/profile
```

### 必填项

#### 1️⃣ 头像（Avatar）

**为什么重要？**
- 📇 别人看到你的第一印象
- 💬 在评论、PR、Issue 中显示
- 🌟 专业的头像有助于建立个人品牌

**建议**：
- ✅ 使用真人照片（建儿）
- ✅ 使用专业的技术相关图片（如代码截图）
- ❌ 避免使用默认头像（看起来像僵尸账号）

**如何上传？**

```
Settings → Profile → Upload new picture
│
选择图片：
├── 建议尺寸：500x500 像素以上
├── 建议格式：PNG 或 JPG
└── 文件大小：小于 1MB
```

---

#### 2️⃣ 个人简介（Bio）

**作用**：简短介绍自己，让别人快速了解你。

**示例**：

```
✅ 好的 Bio：
"Python 开发者 | 开源爱好者 | 喜欢 Rust 和 WebAssembly"

"前端工程师 @腾讯 | Vue.js 贡献者 | 博客：https://..."

❌ 不好的 Bio：
"热爱编程"（太泛）
"123456"（无意义）
```

**如何填写？**

```
Settings → Profile → Bio（输入框）
│
建议包含：
├── 你的技术栈（Python、前端、后端等）
├── 你的公司或学校（可选）
├── 你的个人网站或博客（可选）
└── 你的座右铭（可选，要正能量）
```

---

#### 3️⃣ 所在地（Location）

**作用**：让别人知道你在哪个城市，方便线下聚会。

**建议**：填写城市名，如 `武汉`、`北京`、`Shanghai`。

---

#### 4️⃣ 个人网站（Website）

**作用**：展示你的个人品牌。

**可以填写**：
- 🌍 个人博客（如 `https://yourname.github.io`）
- 💼 LinkedIn 主页
- 🐦 Twitter/X 账号
- 📚 技术专栏（掘金、知乎、CSDN）

---

#### 5️⃣ 社交媒体账号

**支持的平台**：
- Twitter/X
- Mastodon
- LinkedIn

**作用**：让粉丝在其他平台也能关注你。

## 配置安全设置

### ⚠️ 最重要的一步！

GitHub 账号一旦被盗，攻击者可以：
- ❌ 删除你的所有仓库
- ❌ 发布恶意代码
- ❌ 冒充你进行诈骗

### 开启两步验证（2FA）✅

**什么是 2FA？**

```
普通登录：
1. 输入用户名 → 2. 输入密码 → ✅ 登录成功

开启 2FA 后：
1. 输入用户名 → 2. 输入密码 → 3. 输入手机/App 上的验证码 → ✅ 登录成功
```

**为什么需要？**
- 即使密码被盗，攻击者没有你的手机也登录不了！

**如何开启？**

```
Settings → Password and authentication → Two-factor authentication
│
选择验证方式：
├── 📱 Authenticator App（推荐）
│    使用 Google Authenticator、Microsoft Authenticator 等 App
│    每次登录时，App 会生成一个 6 位验证码
│
└── 📲 SMS（不推荐）
     通过短信接收验证码
     缺点：SIM 卡可以被克隆，安全性较低
```

> 📝 **详细步骤**见 [第23章：两步验证](chapter23-2fa-setup.md)

---

### 添加恢复代码（Recovery Codes）

**作用**：如果你的手机丢了，可以用恢复代码登录。

**如何生成？**

```
开启 2FA 时，GitHub 会自动生成恢复代码
│
务必：
├── 📝 下载保存（点击 "Download codes"）
├── 🖨️ 打印出来（放在安全的地方）
└── 🔒 不要泄露给任何人
```

## 配置通知设置

### 为什么要配置？

GitHub 的通知非常多！如果不配置，你的邮箱会被淹没。

### 配置建议

```
Settings → Notifications
```

#### 参与的通知（Participating）

| 通知类型 | 建议 | 说明 |
|----------|------|------|
| **Pull Request reviews** | ✅ 开启 | 有人审查你的 PR 时通知 |
| **Pull Request pushes** | ❌ 关闭 | PR 有新的提交时通知（太频繁） |
| **Issues** | ✅ 开启 | 有人 @ 你时通知 |

#### 订阅的通知（Subscribed）

| 通知类型 | 建议 | 说明 |
|----------|------|------|
| **Comments** | ✅ 开启 | 有人评论时通知 |
| **Pull request reviews** | ✅ 开启 | PR 被审查时通知 |
| **Pull request status updates** | ❌ 关闭 | CI/CD 状态更新（太频繁） |

#### 通知方式

| 方式 | 建议 | 说明 |
|------|------|------|
| **Email** | ✅ 开启 | 重要通知发邮件 |
| **Web** | ✅ 开启 | 在 GitHub 网站上显示通知 |
| **Mobile** | ⚠️ 看情况 | 如果不想被吵，可以关闭 |

## 常见问题

### Q1: 用户名可以改吗？

**A**: ✅ **可以，但要慎重！**

**如何修改？**

```
Settings → Account ownership → Change username
```

**后果**：
- ⚠️ 旧的用户名会被释放（别人可以注册）
- ⚠️ 指向你仓库的旧链接会失效（GitHub 会自动重定向一段时间）
- ⚠️ 如果你已经有了一些 Star，改用户名会让别人找不到你

**建议**：一开始就想好用户名，尽量不要改。

---

### Q2: 可以用中文用户名吗？

**A**: 
- ✅ **技术上可以**（GitHub 支持 Unicode）
- ❌ **但不推荐！**

**原因**：
- 中文用户名在 URL 中会被编码成 `%E4%BD%A0...`，看起来很奇怪
- 有些工具可能不支持中文用户名
- 别人 @ 你的时候，输入中文很麻烦

**建议**：使用拼音或英文名，如 `zhang-san`、`GTX950L`。

---

### Q3: 注册时邮箱可以换吗？

**A**: ✅ **可以，而且推荐！**

**如何修改？**

```
Settings → Emails → Add email address
│
建议：
├── 添加备用邮箱（防止主邮箱丢失）
└── 设置主邮箱（Primary email）
```

**注意**：
- 修改主邮箱后，Git 提交也要相应修改邮箱设置，否则贡献图会不准确。

---

### Q4: 我已经注册了，但想改头像/简介，在哪里改？

**A**: 

```
方法1：访问 https://github.com/settings/profile
方法2：点击右上角头像 → Your profile → Edit profile
```
